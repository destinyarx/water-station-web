# Acceptance Criteria

- [x] Delivery Sales remains the sole featured gradient KPI.
- [x] The other KPI cards show a subtle glow and bottom wave in light and dark mode.
- [x] Today shows only the Yesterday comparison; Yesterday shows only the Today comparison.
- [x] Pending Deliveries never shows a percentage; Week and Month KPI views show none.
- [x] Zero comparison baselines never display `Infinity`, `NaN`, or a misleading percentage.
- [x] Sales versus Expenses defaults to Weekly and its selector does not change other dashboard panels.
- [x] Monthly chart coverage renders one aggregated sales/expense bar pair.
- [x] Low Stock and Maintenance Due use title case and top-right destination buttons.
- [x] A Pending or For Delivery occurrence can be cancelled after a reason is entered.
- [x] Cancelling updates only that delivery row and preserves its parent/future schedule occurrences.
- [x] For Delivery and Completed transitions require a status-specific confirmation.
- [x] Confirmation details identify From Records versus Guest and show concise delivery context.
- [x] The Custom Dates calendar opens in a popover and the trigger always shows the selected count.
- [x] Focused tests, typecheck, lint, full tests, and production build pass.
- [ ] Manual QA covers owner/staff, light/dark, mobile/desktop, keyboard, and populated/empty states.
